# OPC Customer Pipeline

Create a customer pipeline that a solo founder can maintain without CRM overhead.

## Workflow

1. **Define pipeline stages**
   Use these defaults:
   - Target account
   - Researched
   - Contacted
   - Replied
   - Call booked
   - Pain confirmed
   - Offer made
   - Won
   - Lost or nurture

2. **Clarify ICP**
   Capture:
   - Buyer role
   - Urgent pain
   - Current workaround
   - Budget signal
   - Trigger event
   - Bad-fit signal

3. **Design the source mix**
   Choose two channels max:
   - Warm intros
   - Communities
   - LinkedIn or X
   - Review mining
   - Product directories
   - Existing audience
   - Partnerships

4. **Route supporting work**
   - Use `cold-outreach` for outbound sequences.
   - Use `lead-scoring` for prioritization.
   - Use `sales-script` for calls and demos.
   - Use `objection-handling` for objections.
   - Use `review-mining` for voice-of-customer language.

5. **Produce the operating sheet**
   Output a table with columns:
   - Name/account
   - Segment
   - Source
   - Stage
   - Pain hypothesis
   - Next action
   - Date
   - Signal

## Guardrails

- Keep the pipeline simple enough to update daily.
- Follow-up is part of the system, not a separate task.
- Mark lost reasons precisely; they are product and positioning data.
