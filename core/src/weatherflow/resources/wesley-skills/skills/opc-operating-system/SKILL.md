# OPC Operating System

Build the smallest operating system that helps a one-person company make better decisions every week.

## Workflow

1. **Clarify the business stage**
   - Idea validation, pre-revenue, first customers, repeatable sales, or scaling.
   - Ask for product, ICP, current revenue, runway, active channels, and current bottleneck.

2. **Create the control panel**
   Use a compact table with these rows:
   - North-star outcome
   - Revenue metric
   - Customer pipeline metric
   - Product activation metric
   - Content or acquisition metric
   - Cash/runway metric
   - This week's constraint

3. **Route to supporting skills**
   - Use `opc-weekly-review` for weekly execution.
   - Use `opc-customer-pipeline` for discovery, leads, and sales follow-up.
   - Use `opc-offer-sprint` when the offer is unclear or conversion is weak.
   - Use `opc-automation-map` when manual work blocks growth.
   - Use imported skills such as `pricing-strategy`, `unit-economics`, `landing-page`, `copywriting`, `content-strategy`, `cold-outreach`, `churn-analysis`, and `financial-modeling` when the task matches.

4. **Choose one operating cadence**
   Default cadence for OPC:
   - Monday: pick one growth bet, one product bet, one operations bet.
   - Daily: update customer pipeline and ship one visible artifact.
   - Friday: review metrics, write decisions, choose next week's constraint.

5. **Produce durable artifacts**
   Output these sections:
   - Operating snapshot
   - Metrics table
   - Current bottleneck
   - Next seven days
   - Decisions log
   - Risks and deferred work

## Guardrails

- Keep the system smaller than the founder's ability to maintain it.
- Prefer one metric per function over dashboard sprawl.
- Separate facts, assumptions, and guesses.
- Do not invent business data; mark missing data explicitly.
